## 3. Valve Configuration

### 3.1 Admin Valves

#### Retained from v4-5 (unchanged)
- `ANTHROPIC_API_KEY: str` -- API key
- `REQUEST_TIMEOUT: int` -- default 300s
- `ENABLE_WEB_SEARCH: bool` -- default True
- `WEB_SEARCH_MAX_USES: int` -- default 5
- `WEB_SEARCH_DOMAIN_ALLOWLIST: str` -- comma-separated
- `WEB_SEARCH_DOMAIN_BLOCKLIST: str` -- comma-separated
- `ENABLE_CODE_EXECUTION: bool` -- default False
- `ENABLE_SKILL_XLSX/PPTX/DOCX/PDF: bool` -- all default True
- `CUSTOM_SKILL_IDS: str` -- comma-separated
- `ENABLE_PROMPT_CACHING: bool` -- default True
- `CACHE_TTL: str` -- "5min" or "1hour"
- `CACHE_SYSTEM_PROMPT: bool` -- default True
- `CACHE_USER_MESSAGES: bool` -- default True
- `SHOW_WEB_SEARCH_DETAILS: bool` -- default True
- `SHOW_CITATIONS: bool` -- default True
- `SHOW_TOKEN_USAGE: bool` -- default True
- `LOG_LEVEL: str` -- default "DEBUG"

#### Modified from v4-5
- `DEFAULT_MAX_TOKENS: int` -- **changed default from 8192 to 16384**
- `DEFAULT_TEMPERATURE: float` -- **changed default from 1.0 to 1.0** (but now validated: blocked when thinking is active)

#### Removed from v4-5
- ~~`ENABLE_EXTENDED_THINKING: bool`~~ -- adaptive thinking is always-on
- ~~`THINKING_BUDGET_TOKENS: int`~~ -- replaced by effort parameter

#### New in v4-6
- `EFFORT_LEVEL: str` -- `"low"` / `"medium"` / `"high"` / `"max"`, default `"high"`. Controls inference effort via `output_config.effort`.
- `ENABLE_WEB_FETCH: bool` -- default True. Enables the web fetch tool for reading full page content.
- `WEB_FETCH_MAX_USES: int` -- default 10. Max fetches per request.
- `ENABLE_FAST_MODE: bool` -- default False. Enables 2.5x faster inference at 6x cost.
- `ENABLE_COMPACTION: bool` -- default True. Enables automatic conversation compaction for long contexts.
- `COMPACTION_TRIGGER_TOKENS: int` -- default 100000. Token threshold to trigger compaction.
- `SHOW_COST_ESTIMATE: bool` -- default True. Show estimated cost in token usage section.

### 3.2 User Valves

#### Retained from v4-5
- `ENABLE_MY_WEB_SEARCH: bool` -- default True
- `ENABLE_MY_CODE_EXECUTION: bool` -- default False

#### New in v4-6
- `MY_EFFORT_LEVEL: Optional[str]` -- `None` (use admin default), `"low"`, `"medium"`, `"high"`, `"max"`. Per-user effort override.
- `ENABLE_MY_FAST_MODE: Optional[bool]` -- `None` (use admin default). Per-user fast mode toggle.

def _resolve_effort(self, user_valves) -> str:

## 8. Async HTTP

Replace synchronous `requests` with `aiohttp` for non-blocking streaming.

```python
import aiohttp

async def stream_response(self, url, headers, payload, user_valves, __event_emitter__):
    async with aiohttp.ClientSession() as session:
        async with session.post(url, json=payload, headers=headers,
                                timeout=aiohttp.ClientTimeout(total=self.valves.REQUEST_TIMEOUT)) as resp:
            if resp.status != 200:
                # handle error
                return

            async for line in resp.content:
                line = line.decode("utf-8").strip()
                if not line or not line.startswith("data: "):
                    continue
                data = line[6:]
                if data == "[DONE]":
                    break
                event = json.loads(data)
                # process event...
```

This unblocks the event loop, improving OpenWebUI's concurrency under load.

## 11. Cost Estimation

```python
def _format_token_usage(self, usage, is_fast_mode=False):
    # ... existing token display ...

    if self.valves.SHOW_COST_ESTIMATE:
        multiplier = 6.0 if is_fast_mode else 1.0
        input_cost = usage.get("input_tokens", 0) / 1_000_000 * 5 * multiplier
        output_cost = usage.get("output_tokens", 0) / 1_000_000 * 25 * multiplier
        cache_read_cost = usage.get("cache_read_input_tokens", 0) / 1_000_000 * 0.50
        cache_write_cost = usage.get("cache_creation_input_tokens", 0) / 1_000_000 * 6.25
        total = input_cost + output_cost + cache_read_cost + cache_write_cost
        output += f"- **Est. Cost:** ${total:.4f}\n"
```
